## E-Commerce Data Analytics Pipeline | Global Superstore 2016

This project showcases a robust **End-to-End Data Engineering Pipeline** built with Python to process, clean, and consolidate e-commerce data. By automating the extraction and transformation of the **Global Superstore 2016** dataset, this tool prepares raw data for advanced Machine Learning models and high-level Business Intelligence dashboards.

---

## 📌 Project Overview

In real-world e-commerce scenarios, data is often siloed across different sheets or CSV files. This pipeline serves as a centralized solution to:
* **Ingest** multi-source data from Excel sheets (Orders, Returns, and People).
* **Merge** disparate datasets using relational joins to create a unified Master Data Frame.
* **Clean & Standardize** data by handling missing values and correcting data types.
* **Perform Exploratory Analysis** to derive immediate business insights like regional sales performance and return rates.

---

## 🛠️ Tech Stack

* **Language:** Python 3.x
* **Libraries:** * `Pandas`: For high-performance data manipulation and merging.
    * `Matplotlib` & `Seaborn`: For initial data distribution visualization.
    * `Openpyxl`: To handle complex `.xlsx` file structures.

---

## 🚀 Key Features

* **Automated Data Merging:** Executes complex Left Joins on `Order ID` and `Region` to connect customer orders with return status and regional managers.
* **Intelligent Data Cleaning:** * Automatically handles null values in return records (filling with 'No').
    * Standardizes time-series data by converting strings to `datetime` objects.
* **Error Handling:** Includes `try-except` blocks to manage file path issues and unexpected data corruption.
* **BI-Ready Export:** Generates a structured `Cleaned_Global_Superstore.csv` optimized for tools like **Power BI** or **Tableau**.

---

## 📊 Sample Insights Generated

Upon execution, the pipeline provides immediate summary statistics:
* **Regional Sales Performance:** Identifies the top 5 regions contributing to revenue.
* **Operational Health:** Calculates the global Return Rate percentage to monitor product satisfaction.

## ⚙️ Installation & Usage

1. **Clone the repository:**
   ```bash
   git clone https://github.com/abdulrahmanshariff1286-rgb/Retail-Engine-Pipeline.git
   ```

2. **Install dependencies:**
   ```bash
   pip install pandas matplotlib seaborn openpyxl
   ```

3. **Configure Path:**
   Update the `excel_file` variable in the script to point to your local dataset.

4. **Run the Pipeline:**
   ```bash
   python data_cleaning.py
   ```
